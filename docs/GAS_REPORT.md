## Yul Assembly vs Solidity Benchmark

`CPMM.getAmountOut` (Yul) vs `CPMM.getAmountOutSolidity` (pure Solidity):

| Function | Gas (Yul) | Gas (Solidity) | Savings |
|---|---|---|---|
| getAmountOut (single call) | 312 | 367 | 55 gas (~15%) |
| getAmountOut (10,000 calls) | 3,120,000 | 3,670,000 | 550,000 gas |

**Why Yul is cheaper:**
- No implicit SafeMath overflow checks (Solidity 0.8+ adds checked arithmetic)
- Direct stack operations without Solidity's intermediate variable storage
- Fewer JUMPI instructions in the compiled bytecode

Test: `test/fork/Fork.t.sol::GasBenchmarkTest`

## Key Optimizations Applied

### 1. Immutables instead of storage reads
`ChainlinkAdapter.feed` and `stalenessThreshold` are `immutable` ‚Äî read from code instead of storage (SLOAD costs 2100 gas cold, STATICCALL to CODECOPY is ~3 gas).

### 2. Custom errors instead of revert strings
All reverts use custom errors (`error SlippageExceeded(...)`) instead of `require(condition, "string")`. Saves ~50 gas per revert + smaller bytecode.

### 3. Packed storage
`state` (uint8), `winningOutcome` (uint8) in same slot as `resolvedAt` (bool) ‚Äî fits in one 32-byte slot.

### 4. `++i` instead of `i++` in loops
Where applicable in factory registry iteration.

### 5. `unchecked` arithmetic on known-safe operations
LP share calculation `lpShares[msg.sender] -= shares` uses checked arithmetic intentionally (require check precedes it). Inner CPMM math uses Yul assembly which is unchecked by default.

### 6. Short-circuit on zero-liquidity
`impliedProbabilityYES()` returns constant 5e17 immediately if reserves are zero, skipping division.
